# Markdown WYSIWYG

Markdownファイルをその場で直接WYSIWYG編集できるVS Codeのカスタムエディタ。

## 機能

- `.md`ファイルを開いた際、コマンドから見出し・太字・表・コードブロックなどがそのまま整形されて見えるWYSIWYGビューに切り替え可能。
- 編集内容は通常のテキストドキュメントと双方向に同期するため、他の拡張機能(Git、Lintなど)やVS Code標準のテキスト編集機能と共存できる。
- Milkdown(CommonMark + GFM)ベースのため、表・取り消し線・タスクリストなどGFM記法にも対応。
- 見出し一覧を表示するアウトラインパネルを搭載。クリックで該当箇所へジャンプできる。
- ダーク/ライトの配色切り替えに対応。
- 読みやすさに特化したリーディングスタイルを搭載(下記)。

## 表示スタイル

エディタ左上のボタンで表示を切り替えられる。選択状態は保存され、次回開いたときも維持される。

| ボタン | 機能 |
|---|---|
| `☰` | アウトラインパネルの表示切り替え |
| `☽` / `☀` | ダーク/ライトの配色切り替え |
| `Aa` → `明` → `ゴ` | 表示スタイルの循環切り替え |

表示スタイルは3種類ある。

- **エディタ表示**(`Aa`) — ブロック種別ラベル(H1/P/QUOTEなど)を左余白に表示する執筆向けの画面。
- **リーディング明朝**(`明`) — 読み物向けスタイル。明朝体・広い行間・中央寄せの読みやすい幅のカラムで、ブログ記事のように表示する。
- **リーディングゴシック**(`ゴ`) — 技術文書向けスタイル。ゴシック体で、h1/h2の下罫線と見出し直後の詰めによりセクションを追いやすくしている。

いずれのスタイルもダーク/ライト両方の配色に対応する。

## 使い方

1. `.md`ファイルを開く(デフォルトでは通常のテキストエディタで開く)。
2. コマンドパレット(`Cmd/Ctrl+Shift+P`)から以下のいずれかを実行する。
   - **Markdown WYSIWYG: Markdown WYSIWYGで開く** — 現在のファイルをWYSIWYGエディタで開き直す。
   - **Markdown WYSIWYG: Markdown WYSIWYGとテキスト表示を切り替え** — WYSIWYG表示とテキスト表示を相互に切り替える。

動作確認用に、様々なMarkdown記法を含む `sample.md` を同梱している。

## 必要環境

- VS Code `1.85.0` 以上

## プライバシー

このエディタはローカルで完結して動作する。外部への通信やテレメトリの送信は一切行わない。

## 既知の制限

- 生のHTMLブロックなど、一部のMarkdown記法はWYSIWYG編集に完全対応していない場合がある。
- 非常に大きなファイルではエディタの反応が遅くなることがある。
